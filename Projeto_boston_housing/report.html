Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente da determinao para um modelo uma estatstica til em anlise regressa, como se ele frequentemente descrevesse como "good" a capacidade do modelo de fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 7 - Busca em Matriz¶

O que a tcnica de busca em matriz (grid search) e como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Resposta: A tcnica de busca em matriz basicamente um algoritmo que realiza uma serie de buscas para encontrar qual o melhor parmetro que se encaixa em determinado algoritmo de aprendizagem, o paramentro a ser buscado predefinido pelo usuario, a busca realizada de forma exaustiva atraves de um subconjunto especificado manualmente. A tecnica pode ser utilizada para determinar qual os melhores paramentros que podemos utilizar para classificar dados, o melhor parametro descoberto a partir de uma metrica estabelecida, como por exemplo o de validao cruzada o parametro que consiga o melhor score de validao assim definido como o melhor parametro para o nosso algoritimo.

Questão 8 - Validação Cruzada¶

O que a tcnica de treinamento de validao-cruzada k-fold? Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?
Dica: Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada?

Resposta: A tcnica de treinamentode validao-cruzada k-fold funciona da seguinte forma, dividimos a base de dados em k partes e fazemos uma iterao (repetio) k vezes (k a quantidade de partes em que divididos nossos dados), em cada iterao uma parte (um k) ser utilizada para validao, enquanto as demais partes (k-1) so utilizadas para treinamento, a cada nova iterao a parte que utilizamos para validao alterada, de forma que no final das iteraes todas as partes tenham sido utilizadas para fazer a validao 1 vez e as demais treinamento, aps isto calculado a acuracia sobre os erros encontrados em cada iterao.

O que pode dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada que no conseguiremos calcular o quo eficiente um parametro do nosso modelo e claro os beneficios desta tecnica para busca em matriz que assim temos o conjunto de dados de validao que ser responsavel calcular a performance dos parametros gerados e conseguiremos saber o quo eficiente o nosso parametro otimizado.

# TODO: Importar 'make_scorer', 'DecisionTreeRegressor' e 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': list(range(1,11))}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(regressor, params, scoring = scoring_fnc, cv = cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
